═══════════════════════════════════════════════════════════════════════════════
  QUICK COMMANDS (10)
═══════════════════════════════════════════════════════════════════════════════

1.  "Run all tests"
    → Executes npm test (runs all 219 tests across 14 suites)

2.  "Check for errors" or "Lint check"
    → Runs npm run lint (checks ESLint compliance for code quality)

3.  "Security scan" or "Check vulnerabilities"
    → Runs npm audit (checks for dependency vulnerabilities)

4.  "Validate data files" or "Check JSON"
    → Validates all JSON files in data/ directory using validators

5.  "Python check" or "Check GPU"
    → Verifies Python version and checks CUDA availability

6.  "Full system audit" or "Health check"
    → Complete 10-step comprehensive health check with 0-100 score

7.  "What's uncommitted?" or "Git status"
    → Shows git status (uncommitted changes, branch info)

8.  "Run coverage report"
    → Executes npm test with coverage report generation

9.  "Check dependencies"
    → Lists all installed npm dependencies (npm list --depth=0)

10. "What's the last commit?"
    → Shows last git commit (git log -1)


═══════════════════════════════════════════════════════════════════════════════
  DEEP INSPECTION COMMANDS (7)
═══════════════════════════════════════════════════════════════════════════════

11. "Audit all IPC handlers"
    → Searches main.cjs for ipcMain.handle() and verifies preload.js exposures

12. "Check for hardcoded secrets"
    → Grep search for API_KEY, SECRET, PASSWORD patterns in codebase

13. "Find unused variables"
    → Runs ESLint with unused-vars rule to find dead code

14. "Check for TODO comments"
    → Grep search for TODO, FIXME, HACK comments across all files

15. "Verify all imports work"
    → Checks for missing dependencies and broken require() statements

16. "Find memory leaks"
    → Checks for event listeners without cleanup (potential leaks)

17. "Security review"
    → Runs sanitization checks and validates input handling patterns


═══════════════════════════════════════════════════════════════════════════════
  MAINTENANCE COMMANDS (5)
═══════════════════════════════════════════════════════════════════════════════

18. "Update dependencies"
    → Runs npm update and checks for breaking changes

19. "Clean build artifacts"
    → Removes node_modules and coverage, then rebuilds

20. "Reset to clean state"
    → Checks git status and stashes uncommitted changes if needed

21. "Optimize bundle size"
    → Analyzes dependencies and suggests removals to reduce bundle

22. "Check for outdated docs"
    → Compares copilot-instructions.md to actual codebase for accuracy

  SPECIAL AUTOPILOT COMMAND
═══════════════════════════════════════════════════════════════════════════════

"Run comprehensive health check" or "Full system audit"
    → Executes ALL 10 steps automatically:
       1. Code Quality Scan (lint + tests + coverage)
       2. Security Audit (npm audit + auto-fix)
       3. Data Integrity Check (validate all JSON files)
       4. Python Environment Check (version + CUDA + syntax)
       5. File System Validation (verify critical files exist)
       6. IPC Channel Verification (main ↔ preload sync)
       7. Git Status Check (uncommitted changes + last commit)
       8. Performance Metrics (if app running)
       9. Documentation Sync Check (copilot-instructions.md accuracy)
      10. Generate Report (health score 0-100 + recommendations)
